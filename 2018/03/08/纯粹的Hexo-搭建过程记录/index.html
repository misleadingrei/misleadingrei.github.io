纯粹的Hexo-搭建过程记录

使用Hexo的原因

第三方的博客总是有各种各样的限制，不妨使用Hexo对于自己想要的模板与样式进行个性化定制

必要的准备工作

这里默认大家都有了github，没有的话可以去世界上最大的同性交友网站去注册一个，因为要基于github来托管自己的文章，所以需要提前新建一个github的repository，这里对于仓库的名字是有限制的，必须是github.你的github用户名字.io

流程

安装node.js

本人在Centos下使用的hexo，因为hexo是基于node.js的静态博客，于是必要依赖是node.js，去官网选择最新的稳定版本就好了(LST)
点这里
linux下的安装稍微有点不一样，参考
如果在bash下使用node -v 跳出版本号，那么就安装成功了

安装Hexo

新建一个你想要保存的日志文件夹，之后我们的日志和hexo文件都会保存在这里，cd 到这个日志文件夹 ，执行 npm i -g hexo 进行安装，同样的，使用hexo -v进行安装验证。安装成功之后执行hexo init 进行初始化产生必要文件。

配置SSH

如果有ssh免密码登录的需求的话，可以考虑使用SSH方式来进行推送登录，否则则可以考虑http用户名和密码登录。首先是必要的SSH公私钥的产生过程使用

ssh-keygen -t rsa -C “youremail@example.com“

产生公私钥，默认情况下公私钥的位置是～/.ssh/
使用

cat ～/.is_rad.pub

显示自己的公钥，将其复制到github的公钥上去。
使用

ssh -T git@github.com

来测试SSH是否链接成功，如果成功的话，则会出现

You’ve successfully authenticated, but GitHub does not provide shell access.

Hexo的本地部署

修改自己的博客目录下的（亦即是Hexo目录下的_config.yml配置文件）来完成部署，注意此时有
坑一号
yml文件对于格式是有具体要求的，如果格式错误的话会导致deploy过程无法成功。可以参考知乎此问具体的要求是

冒号之后必须是有一个半角空格

如果使用SSH

deploy:
type: git
repo: git@github.com:YourgithubName/YourgithubName.github.io.git
branch: master

如果使用https

deploy:
type: git
repo: https://github.com/YourgithubName/YourgithubName.github.io.git
branch: master

配置完成之后，可以在本地端口进行本地端的预部署，由于hexo在3.0之后把服务器独立成个别模块，需要单独安装：npm i hexo-server。
之后执行：

hexo clean
hexo generate
hexo server

打开localhost:4000,4000是Hexo的默认端口，当然也可以通过Hexo s -p 来指定端口。在某些情况下会出现
坑二
打开localhost：4000的时候是有概率没有响应的，这时有大概率是端口被占用了，而且这个概率在window下还是挺高的哈哈哈哈，因为有个国内的PDF厂商阅读器的某个进程一直占着这个端口参考，这个时候可以考虑换端口，也可以考虑kill掉这个进程。
如果没有问题的话应该可以看到Hexo的页面了。

Hexo的上传部署

执行

npm install hexo-deployer-git –save

来保存hexo的git部署器。
安装完成之后
持行：

hexo clean
hexo generate
hexo deploy

ok,可以去github上找自己的博客咯

http://yourgithubname.github.io

个性化定制

可以去Hexo自己的主题仓https://hexo.io/themes/
或者是看看有人爬的结果
https://www.zhihu.com/question/24422335
更换主题很简单，只需要修改_config.yml配置文件下theme对应的主题配置就好了

Extensions
Plugins: https://hexo.io/plugins/
Themes: https://hexo.io/themes/
theme: landscape-plus

配置完以后重新genarate 和delpoy就好了
